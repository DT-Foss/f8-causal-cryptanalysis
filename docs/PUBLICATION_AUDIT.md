# Publication audit

## Scope

This repository is a curated publication copy. The active laboratory tree was
treated as read-only. Publication edits, metadata sanitization, documentation,
CI, and Git history were created only in the independent
`f8-causal-cryptanalysis` tree.

## Included scientific material

- installable F8, CASI/LiveCASI, CryptoCausal, cipher, and CLI code;
- tests and frozen experiment configurations;
- A001--A209 experiment sources, frozen protocols, and append-only attempt ledger;
- retained result JSON, `.causal` graphs, reports, and hash manifests;
- A207's 11,145,296-byte exact structural-order archive, frozen protocol,
  complete 352-cell result, final Causal graph, progress map, and tests;
- A208's complete 32-cell long-budget phase map and A209's complete 256-cell
  Width-12 phase-reset composition, with configs, runners, tests, reports,
  result JSON, and Causal graphs;
- the committed twelve-configuration F8 anchor suite exported from source
  commit `2e23b23e46cd7a413edd5b56a748e4d5e6e62f73`;
- Nanjing/Rome reference datasets and author-owned TeX;
- sanitized author-owned Nanjing PowerPoint;
- the minimum third-party source file required by the SHA-2 experiment.

| `.venv`, caches, `__pycache__`, egg-info, build and distribution outputs | generated local state |

The PQCrypto exclusion retains one required file,
`pqclean/common/sha2.c`, together with upstream Apache-2.0 `LICENSE` and
`NOTICE`. The author-owned legacy LiveCASIv2 dependency remains as a 42-KB
importable source bundle because the historical replay runner hashes and imports
that exact bundle.

## Sanitization performed

### Host paths

Historical run records contained an author's absolute home path in command,
working-directory, interpreter, traceback, and dependency fields. Only those
path strings were replaced with stable role labels such as `<repository>` and
`<legacy-live-casi>`. Numeric stdout, seeds, timestamps, return codes, and source
hash fields were retained. The reproduction index and its SHA-256 manifest were
regenerated after sanitization.

The legacy runner itself now emits role paths and sanitizes captured tracebacks,
so a future `--force` replay does not reintroduce private paths.

### PowerPoint metadata

The Nanjing deck's title, subject, creator, and last-modified-by fields were
normalized to the public work and `David Tom Foss`. Slide XML, relationships,
and media were not edited. ZIP integrity and canonical creator fields are
checked in CI.

### Author identity

README, CFF, package metadata, paper sources, presentation metadata, local Git
author/committer configuration, tag, and release text use **David Tom Foss**.
The commit e-mail `david@foss.com.de` was taken from the author's existing
public anchor commit; no address was invented.

## Automated publication gates

`scripts/check_publication.py` fails on:

- home-directory absolute paths;
- private-key blocks and distinctive GitHub/OpenAI/AWS token forms;
- non-canonical public author forms;
- broken relative Markdown links;
- symlinks or individual files above 50 MiB;
- missing publication ledgers or manifests;
- a corrupt presentation or non-canonical deck creator metadata.

At this A185--A209 publication candidate, the audit reports zero secret matches, zero
private absolute paths, zero broken relative links, zero symlinks, zero files
above 50 MiB, and zero non-canonical author identities. The audited tree at
this stage contains 1,544 files, of which 1,131 are UTF-8 text inputs, totaling
107,032,937 bytes before Git object compression. The largest retained
object is the 11.1 MB A207 NumPy order archive, below both the repository's
50 MiB audit ceiling and GitHub's per-file limit.
