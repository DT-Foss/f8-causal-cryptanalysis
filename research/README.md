# Research evidence workspace

This directory contains the reproducible claim audit and follow-up experiments
for the F8 and CASI work. It deliberately separates claims, executable
experiments, raw results, and interpretation.

## Layout

- `claims/`: one row per material paper claim, with exact evidence paths.
- `configs/`: immutable experiment parameters.
- `experiments/`: executable research code; no paper prose is generated here.
- `results/`: raw machine-readable outputs and run manifests.
- `reproduction/`: frozen inventory, code-version protocol, and legacy runner.
- `reports/`: reviewed interpretation of the raw outputs.
- `ATTEMPT_LOG.md`: every attempted experiment, including negative or failed runs.

The unnumbered formula-atlas full re-audit is implemented by
`experiments/formula_atlas_transfer_audit.py`, retained as
`results/v1/formula_atlas_transfer_coverage_v1.json`, and interpreted in
`reports/FORMULA_ATLAS_FULL_REAUDIT_V1.md`.  It reads all 2,411 atlas entries
across nine papers and 113 source pages without a keyword prefilter, drops no
entry, and registers 18 controlled transfer families.  These are hypotheses
with explicit invariants, not cryptanalytic results.

## Reproduction

From the repository root:

```bash
./scripts/reproduce_papers.sh
./scripts/reproduce_research.sh
```

The PQC track has a separate real-backend runner:

```bash
./scripts/reproduce_pqc.sh
```

It pins `pqcrypto==0.4.0`, functionally gates every KEM/signature operation,
then runs the output controls, the ML-KEM NTT trace, and the ML-KEM packing
isolation. It also runs a balanced BvN-route causal gate: global bijections
preserve every 32-byte output row exactly while changing only row order. This
separates output-order/serialization effects from claims about internal PQC
state. Its output hash manifest is `results/v1/PQC_SHA256SUMS`.

The route construction and its calibration are documented in
`reports/BVN_ROUTE_CAUSAL_V1.md`. Use it for any new sequential PQC statistic
before interpreting an elevated CASI or F8-O score.

The operation-level causal track has its own runner:

```bash
./scripts/reproduce_causal_mechanisms.sh
```

It validates the exact cryptographic `.causal` builder/reader, runs paper-scale
carry interventions over all F8 targets, checks the corrected ML-KEM
`Uniform(Z_q) -> Compress_d` cause, evaluates six lossless compressor families
and all ordered depth-two cascades, and computes the conditional code-length
form of F8. The reviewed interpretation is
`reports/CAUSAL_MECHANISM_RESULTS_V1.md`; all retained artifacts are pinned in
`results/v1/CAUSAL_SHA256SUMS`.

Every `.causal` artifact is an exact, typed cryptographic evidence graph, not
an opaque sidecar.  `CryptoCausalReader` checks the CAUSAL header/version,
canonical graph digest, triplets, and inferred-edge provenance.  New AES runs
perform this read-back immediately after writing; the full repository gate is:

```bash
.venv/bin/python scripts/validate_causal_artifacts.py
```

The format stores machine-readable parameters, evidence kind, source,
confidence, attributes, and provenance inside the file.  It deliberately does
not claim to contain unstructured LLM chain-of-thought; the causal content is
the typed, auditable reasoning graph itself.

The second command is fail-closed: it starts follow-up work only after all 39
historical F8 experiments and the Nano master rebuild have completed.

Paper reproduction is performed first with the read-only legacy runner described
in `reproduction/REPRODUCTION_PROTOCOL.md`. Follow-up results under `results/v1/`
are preliminary until that baseline closes. Corrected primitive reruns are kept
separate from historical replay.

## Evidence statuses

- `EVIDENCED`: script, retained raw result, and exact reproduction command exist.
- `PARTIAL`: at least one of those three artifacts is missing or mismatched.
- `MISSING`: no local executable/data evidence was found.
- `CONTRADICTED`: retained evidence directly disagrees with the claim.
- `EXTERNAL_ONLY`: support depends on a cited external primary source.

No security claim is promoted from an experiment merely because a p-value is
small. The relevant cryptographic security game and ideal-world control must be
stated separately.
